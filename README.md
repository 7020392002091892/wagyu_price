# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
リモートリポジトリ：ネット上に配置して複数人で共有するためのリポジトリ
ローカルリポジトリ：開発者１人１人が使用するために自分のPC上に配置するためのリポジトリ

## プッシュとマージの違いは何でしょうか？
プッシュ：ローカルリポジトリのコミット（変更履歴）をリモートにアップロードする
マージ　：異なるブランチで行われた作業を統合する　ローカル内での操作のため、さらにPushするまではリモートに反映されない。



## コミットとプッシュの違い
コミット：ローカルで変更したデータを自分のPC上に履歴を残す動作
プッシュ：ローカルに記録されたデータをリモートに送信する動作

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
どのような修正点・追加点を行ったかを明記してあげる。


## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
マージを行うタイミングが、ローカルではリモートにプッシュする前にマージが行われているのに対し、プルリクエストでは変更点をリモート側へ一度プッシュし、レビュー依頼を行ったうえで、最後にリモート側でマージを行っている。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？
コンフリクト＝他の人も同じソースを修正し競合が発生した。
１．競合が発生したことを上長に相談し自分１人での解決をしないようにする。
２．もし、自分で解決することになった場合は
どちらを正にするかや、どちらも実装すべきかを同じソースを修正した人と相談しながら修正を行う。
